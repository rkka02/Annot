# Annot : PDF Research Assistant — 프로젝트 계획서


## 1. 프로젝트 개요

### 목적
논문을 읽으며 동시에 AI와 대화할 수 있는 도구를 만든다.
VSCode가 "코드 편집기 + AI 채팅"의 조합이라면, 이 프로젝트는 **"PDF 리더 + AI 채팅"의 조합**이다.

### 해결하려는 문제
- 현재 ChatGPT에 PDF를 업로드하면, PDF 원문과 채팅창을 동시에 볼 수 없다.
  원문을 확인하려면 앱을 왔다갔다 해야 하고, 맥락이 끊긴다.
- 매번 새 대화를 시작하면 이전에 했던 질문과 맥락이 사라진다.
- 논문을 읽다가 떠오르는 질문을 즉시, 흐름이 끊기지 않는 환경에서 던지고 싶다.

### 핵심 가치
PDF를 **보면서** AI에게 질문하고, 그 대화가 **세션으로 누적**되어 다음에 이어서 쓸 수 있는 환경.

## 2. 설계 논의 과정

### 2.1 초기 아이디어
- PDF 뷰어 + AI 채팅 패널이 나란히 배치된 인터페이스.
- 논문 읽으면서 중요한 부분은 노란색 하이라이트, 모르는 부분은 빨간색 하이라이트로 개인 정리.
- 모르는 부분이 있으면 옆 채팅창에 질문.

### 2.2 컨텍스트 범위: 트리 기반 스코핑
- 초기에는 "프로젝트 단위"로 이전 논문, 메모, 대화 이력을 참조하는 구상이 있었다.
- 이후 **트리(폴더) 구조**로 발전. 리서치 자료를 계층적으로 구조화하고, 각 폴더 노드에서 세션을 열 수 있다.
- **핵심 규칙**: 컨텍스트는 아래로만 흐른다.
  - 폴더에서 세션을 열면, 해당 폴더와 모든 하위 폴더의 PDF에 접근 가능.
  - 상위 폴더나 형제 폴더의 PDF는 참조 불가.
  - PDF(리프 노드) 자체에서는 세션을 열 수 없다. 세션은 항상 폴더에서 시작.
  - PDF는 해당 세션 모델이 접근 가능한 **리소스**로 기능한다.

#### 예시
```
My Research/
├── 📁 Machine Learning          ← 세션: survey.pdf + 하위 전체 PDF 참조 가능
│   ├── 📄 survey-2024.pdf
│   ├── 📁 Transformers
│   │   ├── 📄 attention.pdf
│   │   └── 📄 bert.pdf
│   └── 📁 Reinforcement Learning
│       └── 📄 ppo-paper.pdf
└── 📁 Physics                   ← ML 세션에서는 이쪽 접근 불가
    └── 📄 quantum.pdf
```

### 2.3 RAG 관련 논의
- 프로젝트에 자료가 쌓이면 컨텍스트 윈도우에 다 넣을 수 없으므로 RAG이 필요.
- 단, RAG은 파편화된 정보만 가져오고 유기적 맥락은 놓치는 한계가 있음.
- **해결 아이디어**: RAG top-K 결과 + 그래프의 연결된 노드를 함께 가져오는 2단계 검색.
- 이 역시 MVP 이후 단계로 보류. 현재는 트리 스코핑 + 세션 기반 대화 이력으로 컨텍스트를 직접 관리.

### 2.4 AI 호출 방식
- **Codex 백엔드 재사용 방식 채택**: 로컬에 설치된 Codex의 OAuth 토큰(`~/.codex/auth.json`)을 읽어서 OpenAI API를 호출.
- 별도 API 키 불필요. ChatGPT 구독으로 사용. 한 번 Codex에 로그인하면 Annot이 자동으로 인증을 재사용.
- 모델은 `/v1/models` API로 동적 목록을 불러와 사용자가 선택.

### 2.5 플랫폼 결정
- 현재는 Next.js 웹앱으로 개발 (빠른 프로토타이핑).
- **궁극적 목표**: 맥 네이티브 앱 (Tauri 또는 Electron). 맥북에서 로컬로 동작.
- 윈도우 빌드도 가능하지만, 우선순위는 macOS.

### 2.6 데이터 저장: 로컬 파일시스템
- 트리 구조를 로컬 폴더 구조로 그대로 미러링.
- PDF 파일은 실제 파일시스템 폴더에 배치. Finder에서도 직접 확인/관리 가능.
- 세션 이력, 하이라이트 등 메타데이터는 각 폴더에 `.annot/` 숨김 디렉토리로 저장.

```
~/Annot/
├── Machine Learning/
│   ├── .annot/                  ← 메타데이터 (세션 이력, 하이라이트 등)
│   │   ├── sessions.json
│   │   └── highlights.json
│   ├── survey-2024.pdf
│   └── Transformers/
│       ├── .annot/
│       │   └── sessions.json
│       ├── attention.pdf
│       └── bert.pdf
└── Physics/
    ├── .annot/
    │   └── sessions.json
    └── quantum.pdf
```

## 3. MVP 스펙

### 기능
| 기능 | 설명 |
|------|------|
| 트리 탐색기 | 로컬 폴더 구조를 사이드바에 트리로 표시. 폴더/PDF 탐색 |
| PDF 뷰어 | pdf.js 기반. 논문을 브라우저에서 렌더링 |
| 하이라이트 | 노란색(중요), 빨간색(모름) 하이라이트. 개인 정리용 |
| AI 채팅 패널 | 폴더 노드에서 세션을 열고 AI와 대화. 트리 스코프 내 PDF를 리소스로 참조 |
| 세션 관리 | 대화 이력을 폴더별 `.annot/`에 저장. 폴더를 다시 열면 이어서 사용 가능 |
| 컨텍스트 참조 | 해당 폴더 + 하위 폴더의 PDF + 현재 세션의 대화 이력 |
| 모델 선택 | Codex 백엔드에서 사용 가능한 모델 목록을 불러와 동적 선택 |

### 기술 스택
| 구성 요소 | 기술 |
|-----------|------|
| 프론트엔드 | Next.js 웹앱 (→ 추후 네이티브 앱) |
| PDF 렌더링 | pdf.js |
| AI API | OpenAI (Codex 백엔드, OAuth 토큰 재사용) |
| 데이터 저장 | 로컬 파일시스템 (폴더 = 트리, `.annot/` = 메타데이터) |

### 레이아웃
- **좌측 사이드바**: 트리 탐색기 (폴더/PDF 계층 구조)
- **중앙**: PDF 뷰어 (선택한 PDF 렌더링)
- **우측**: AI 채팅 패널 (현재 폴더 세션)
